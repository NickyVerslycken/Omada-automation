Project code map with classes/functions and responsibilities
==========================================================

Scope
- This file documents the active Python code in this repository.
- It lists files, classes, functions/methods, and what each does.

Entry points
- `start-omada-app.pyw`
  - Imports `main` from `runners.run_gui` and starts the GUI app.
- `runners/run_gui.py`
  - `main()`: launches `App().mainloop()`.
- `runners/sync_changelog_from_git.py`
  - `main()`: appends changelog entries for current git worktree changes.

Package: `omada_batch`

- `omada_batch/config.py`
  - Constants plus env helpers.
  - Purpose: central file path config for data/profile storage and `data/.env` loading/upserts.
  - `load_env_file(...)`: parses `data/.env` and injects keys into process env.
  - `upsert_env_file(...)`: creates/updates key-value pairs in `data/.env`.
  - `delete_env_keys(...)`: removes selected keys from `data/.env` and process env.

- `omada_batch/models/controller_profile.py`
  - `ControllerProfile` (dataclass): typed controller profile record.

- `omada_batch/models/lan.py`
  - `PlannedLan` (frozen dataclass): one planned LAN/VLAN unit for preview/push.

- `omada_batch/storage/file_change_log.py`
  - `append_file_change_log(action, target_path, ...)`: appends JSONL file change events with timestamp, action, path, rollback instructions, and optional details.
  - `write_json_with_changelog(target_path, payload, ...)`: writes JSON and logs the change as `created` or `edited`.
  - `delete_file_with_changelog(target_path, ...)`: deletes a file and logs the change as `deleted`.
  - `sync_git_worktree_to_changelog(...)`: scans `git status` and appends changelog entries for created/edited/deleted/moved files (with filters for backups/cache artifacts).
  - Default changelog location is `backup/file_changelog.jsonl`.
  - Path values in changelog entries are normalized to `Omada-automation/...` format with forward slashes.
  - For `edited`/`deleted` entries, rollback details include a concrete `details.backup_path` to a real timestamped backup file when available.

- `omada_batch/storage/profile_store.py`
  - `ProfileStore`
  - `__init__(path=None, changelog_path=None)`: selects default/legacy profile file and optional changelog destination.
  - `load_raw()`: reads and normalizes profile list from JSON.
  - `save_raw(profiles)`: writes canonical profile JSON via changelog-aware writer.

- `omada_batch/services/profile_service.py`
  - `normalize_profile_id(...)`: sanitizes profile IDs for stable JSON/env-key usage.
  - `suggest_profile_id(...)`: derives profile ID seed from profile name/base URL.
  - `ensure_unique_profile_id(...)`: prevents ID collisions by suffixing (`_2`, `_3`, ...).
  - `profile_env_var_names(profile_id)`: builds per-profile env key names (`OMADA_PROFILE_<ID>_*`).
  - `normalize_profile(item, index=0)`: canonicalizes profile keys, resolves credentials from env var references (including ID-based defaults), and preserves legacy plaintext compatibility.

- `omada_batch/services/planner.py`
  - `generate_plan(...)`: validates parameters and builds `PlannedLan` list + warnings.

- `omada_batch/services/lan_service.py`
  - `build_interface_catalog(devices)`: extracts gateway/switch + interface metadata, preferring stable `portId` IDs and natural port `name` labels.
  - Marks WAN ports as non-selectable for LAN assignment and deduplicates repeated interface IDs.

- `omada_batch/services/device_service.py`
  - `_is_generic_interface_name(name, iid)`: detects placeholder names.
  - `_clean_interface_name(name, iid)`: strips noisy id suffixes.
  - `interface_display_name(iface)`: picks best label for UI display.
  - `merge_interface_catalog_names(base, extra)`: enriches weak names while preserving rich interface metadata keys for display.
  - `extract_interface_catalog_from_networks(networks)`: derives interface id/name list from LAN payloads.

- `omada_batch/api/omada_client.py`
  - `OmadaOpenApiClient`
  - `__init__(...)`: initializes HTTP session, token state, and optional raw JSON debug logger callback.
  - `_req(...)`: shared HTTP request helper with validation and request/response JSON debug emission.
  - `_emit_json(...)`: safe JSON-debug callback emitter.
  - `_sanitize_headers(...)`: redacts sensitive headers before debug output.
  - `_preview_text(...)`: truncates long response text for logging/debug output and only appends `...` when actually truncated.
  - `_auth_headers()`: constructs auth header from token.
  - `get_controller_info()`: fetches controller info and `omadacId`.
  - `get_access_token_client_credentials(...)`: obtains access token using fallback payload formats.
  - `get_sites(...)`: site listing (v3/v2/v1 fallback) with explicit success/fail logs.
  - `get_lan_networks(...)`: reads site LAN networks (v3/v2/v1 fallback) with explicit success/fail logs.
  - `get_site_devices(...)`: reads site devices using endpoint/param fallbacks (now includes v3 endpoint priority) and enriches devices with port details.
  - `get_device_port_details(...)`: fetches per-device port labels/IDs via `networks/devices/ports` using v3/v2/v1 fallback and payload retries.
  - `_enrich_devices_with_port_details(...)`: merges port payload data into device records for downstream interface catalog building.
  - `get_site_gateways(...)`: resolves gateway candidates from device payloads (v3/v2/v1 endpoint tries).
  - `create_lan_network(...)`: creates LAN with endpoint fallback (v3/v2/v1) plus request-field retries.

UI package split

- `omada_batch/ui/theme.py`
  - Centralized theme constants and helpers for the modernized UI.
  - Color palette (PRIMARY, SURFACE, TEXT_*, etc.), font definitions, card/button sizing constants.
  - `configure_treeview_style()`: applies modern flat styling to `ttk.Treeview` and `ttk.Combobox` widgets.

- `omada_batch/ui/main_window.py`
  - Thin compatibility shim that re-exports `App` from `ui.shell.app_shell`.

- `omada_batch/ui/state/app_state.py`
  - `AppState` (dataclass): central mutable runtime state container for UI workflows.

- `omada_batch/ui/controllers/connection_controller.py`
  - `ConnectionControllerMixin`: profile CRUD/import/export and controller connect/disconnect/site workflows.
  - Connect flow now wires API raw JSON events into UI queue via `("devjson", payload)`.
  - Save flow assigns/keeps stable profile IDs, writes credentials to `data/.env` under per-profile env keys, and persists only env variable key references in profile JSON.
  - Remove flow now deletes unreferenced per-profile env keys from `data/.env` when a profile is removed.
  - Export flow writes profile JSON through changelog-aware storage helper without raw credentials.

- `omada_batch/ui/controllers/networks_controller.py`
  - `NetworksControllerMixin`: gateway refresh/selection, current networks fetch/filter/export, interface catalog helpers.
  - Filters DHCP server combobox options to gateway devices only (non-gateway devices are skipped).
  - `_network_matches_gateway(...)` now also considers extra Omada ownership fields such as `gatewayMac` and `assignIpDeviceMac`.
  - Gateway selection changes now trigger refresh of inline batch-tab interface mapping controls.
  - Network export now writes JSON through changelog-aware storage helper.

- `omada_batch/ui/controllers/batch_controller.py`
  - `BatchControllerMixin`: plan preview/export and push-to-controller workflow.
  - Builds and maintains inline LAN-interface mapping UI state in Batch Create tab (no popup), rendered as a grid/table.
  - Resolves interface catalog from selected DHCP server, deduplicates interface IDs, and avoids injecting unrelated network-derived IDs.
  - Disables WAN interfaces in selection UI while keeping them visible with explicit WAN-disabled labels.
  - Validates per-network interface selection directly from tab controls before push.
  - Plan export now writes JSON through changelog-aware storage helper.

- `omada_batch/ui/shell/app_shell.py`
  - `App` class (CTk root): composes mixins, owns queue/worker loop, builds sidebar + header + status bar layout.
  - Uses `customtkinter.CTk` as base class instead of `tk.Tk` for modern UI.
  - Left sidebar navigation replaces top `ttk.Notebook` tabs. Pages show/hide via `_show_page(name)`.
  - Header bar shows current page title. Status bar shows connection state and controller info.
  - Developer mode toggle moved to sidebar bottom (CTkSwitch). Dynamically adds/removes dev nav item.
  - Provides property adapters from legacy field names to `AppState` (stored as `self.app_state` to avoid conflict with CTk's `.state()` method).
  - Adds numbered-line append helpers for logs and developer JSON output.
  - Applies 250-character log line truncation with ellipsis only when truncation occurs.
  - `_poll_queue(...)` routes `devjson` queue events to the developer output pane. Progress uses `CTkProgressBar.set()` API.

- `omada_batch/ui/tabs/connection_tab.py`
  - `build_connection_tab(app)`: builds connection page with card-based layout (CTk widgets). Cards: Controller Profiles, Endpoint Configuration, Site Selection.

- `omada_batch/ui/tabs/current_networks_tab.py`
  - `build_current_networks_tab(app)`: builds current-LAN page with CTk toolbar buttons, modern-styled Treeview in a card, and CTkTextbox raw JSON viewer.

- `omada_batch/ui/tabs/batch_tab.py`
  - `build_batch_tab(app)`: builds batch planning page with card-based parameter form, CTk action buttons, CTkProgressBar, interface selection matrix, and modern-styled plan preview Treeview.

- `omada_batch/ui/tabs/logs_tab.py`
  - `build_logs_tab(app)`: builds log page with CTkTextbox (monospace, wrap=none) in a card and CTk clear button.

- `omada_batch/ui/tabs/developer_tab.py`
  - `build_developer_tab(app)`: builds developer JSON page with CTkTextbox (monospace, wrap=none) in a card and CTk clear button.

- `omada_batch/ui/dialogs/interface_selector.py`
  - `prompt_interface_selection(...)`: legacy modal mapping helper using CTkToplevel; batch flow now uses inline tab controls.

- `omada_batch/ui/app.py`
  - Re-export/import shim for `App`.

- `omada_batch/ui/background_tasks.py`
  - Placeholder for shared background helpers.

- `omada_batch/ui/state.py`
  - Legacy placeholder for future state dataclasses.

- `omada_batch/utils/validators.py`
- `omada_batch/utils/formatters.py`
- `omada_batch/utils/logging.py`
  - Placeholder utility modules reserved for shared helpers.
