Project code map with classes/functions and responsibilities
==========================================================

Scope
- This file documents the active Python code in this repository.
- It lists files, classes, functions/methods, and what each does.

Entry points
- `start-omada-app.pyw`
  - Imports `main` from `runners.run_gui` and starts the GUI app.
- `runners/run_gui.py`
  - `main()`: launches `App().mainloop()`.

Package: `omada_batch`

- `omada_batch/config.py`
  - Constants only:
  - `PACKAGE_DIR`, `PROJECT_ROOT`, `DATA_DIR`, `DEFAULT_PROFILE_PATH`, `LEGACY_PROFILE_PATH`.
  - Purpose: central file path config for data/profile storage.

- `omada_batch/models/controller_profile.py`
  - `ControllerProfile` (dataclass): typed representation of a saved controller profile.

- `omada_batch/models/lan.py`
  - `PlannedLan` (frozen dataclass): one planned LAN/VLAN item (name, vlan, subnet, gateway, DHCP range).

- `omada_batch/storage/profile_store.py`
  - `ProfileStore`
  - `__init__(path=None)`: picks default/legacy JSON profile path.
  - `load_raw()`: reads profile JSON and normalizes accepted shapes (`list`, `{"profiles": [...]}`, `{"controllers": [...]}`).
  - `save_raw(profiles)`: writes profiles to JSON (`{"profiles": [...]}`), creating parent folder when needed.

- `omada_batch/services/profile_service.py`
  - `normalize_profile(item, index=0)`: converts loose profile objects into canonical keys and default name.

- `omada_batch/services/planner.py`
  - `generate_plan(...)`: validates user inputs and builds a list of `PlannedLan` items plus warnings.
  - Handles subnet stepping, VLAN incrementing, and DHCP range clamping.

- `omada_batch/services/lan_service.py`
  - `build_interface_catalog(devices)`: extracts gateway/switch devices and LAN interface metadata into UI-ready catalog entries.

- `omada_batch/services/device_service.py`
  - `_is_generic_interface_name(name, iid)`: detects placeholder interface labels.
  - `_clean_interface_name(name, iid)`: strips noisy trailing `[id]` / `(id)` suffixes.
  - `interface_display_name(iface)`: picks best human-readable interface name.
  - `merge_interface_catalog_names(base, extra)`: keeps IDs from `base`, enriches names from `extra` when base names are weak.
  - `extract_interface_catalog_from_networks(networks)`: builds interface id/name catalog from LAN network payloads.

- `omada_batch/api/omada_client.py`
  - `OmadaOpenApiClient`
  - `__init__(...)`: initializes HTTP session/auth state.
  - `_req(...)`: common HTTP request wrapper (headers, body mode, status/JSON checks).
  - `_auth_headers()`: returns `Authorization` header from current token.
  - `get_controller_info()`: calls `/api/info` and stores `omadacId`.
  - `get_access_token_client_credentials(...)`: tries multiple payload formats to obtain token, stores expiry.
  - `get_sites(...)`: lists controller sites (v2 then v1 fallback).
  - `get_lan_networks(site_id, ...)`: reads LAN networks for a site.
  - `get_site_devices(site_id, ...)`: fetches network devices via multi-endpoint/multi-param fallbacks.
  - `get_site_gateways(site_id, ...)`: derives gateway candidates and interface IDs from device list.
  - `create_lan_network(site_id, plan, gateway_device=None, interface_ids=None)`: creates one LAN network with retry fallbacks for gateway/interface and gatewaySubnet variants.

- `omada_batch/ui/dialogs/interface_selector.py`
  - `prompt_interface_selection(parent, plan, interfaces, display_name_fn)`:
  - Opens modal UI to select one or more interfaces per planned network.

- `omada_batch/ui/main_window.py`
  - `App` (Tk root window): main GUI orchestration and application state.
  - Lifecycle/core:
  - `__init__()`: initializes state, loads profiles, builds UI, starts queue polling.
  - `_build_ui()`: creates notebook and tab frames.
  - `_poll_queue()`: consumes background-worker events (`log`, `error`, `sites`, `gateways`, `networks`, `progress`, `info`).
  - `_run_bg(work, disable_buttons=...)`: runs long work in thread and coordinates button states.
  - Logging/UI state:
  - `_build_logs_tab()`, `_log()`, `on_clear_log()`.
  - Connection/profile flow:
  - `_build_connection_tab()`, `_load_controller_profiles()`, `_save_controller_profiles()`, `_normalize_controller_profile()`, `_controller_host_key()`, `_refresh_controller_profile_combo()`, `_selected_controller_profile_index()`, `on_controller_profile_selected()`, `on_save_controller_profile()`, `on_remove_controller_profile()`, `on_import_controller_profiles()`, `on_export_controller_profiles()`, `on_connect()`, `_on_connected()`, `on_disconnect()`, `on_refresh_sites()`, `_on_sites()`, `on_site_selected()`.
  - Gateway/device flow:
  - `on_refresh_gateways()`, `_build_interface_catalog()`, `_on_gateways()`, `_current_gateway()`, `_batch_gateway()`, `_batch_gateway_name()`, `on_gateway_selected_current()`, `on_gateway_selected_batch()`.
  - Current networks tab flow:
  - `_build_current_tab()`, `on_fetch_networks()`, `_on_networks()`, `_apply_network_filter()`, `_on_network_selected()`, `on_export_networks()`.
  - Batch planning/push flow:
  - `_build_batch_tab()`, `on_generate_preview()`, `on_export_plan()`, `on_push_plan()`, `_update_push_state()`.
  - Helper relations for interface/network matching:
  - `_network_matches_gateway()`, `_interface_display_name()`, `_merge_interface_catalog_names()`, `_prompt_interface_selection()`, `_get_interface_catalog_from_networks()`.
